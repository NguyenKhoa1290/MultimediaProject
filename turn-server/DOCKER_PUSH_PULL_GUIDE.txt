# 🔄 QUY TRÌNH ĐỒNG BỘ TURN SERVER (LOCAL -> DOCKER HUB -> CLOUD)

Để các tính năng mới (như theo dõi lưu lượng theo phòng) hoạt động, bạn cần đồng bộ mã nguồn local lên Image trên Cloud.

## Bước 1: Build và Push từ máy LOCAL (Nơi bạn sửa code)
Mở Terminal tại thư mục 'turn-server' trên máy tính của bạn và chạy:

```bash
# 1. Login vào Docker Hub (Nếu chưa login)
docker login

# 2. Build lại image mới với tag latest
docker build -t nguyen13104/my-turn-server:latest .

# 3. Đẩy (Push) bản mới lên Docker Hub
docker push nguyen13104/my-turn-server:latest
```

---

## Bước 2: Cập nhật trên máy CLOUD (Server chạy ứng dụng)
Sau khi đã push thành công ở Bước 1, bạn quay lại máy Cloud và chạy:

```bash
# 1. Dừng và xóa container cũ
docker stop turn-srv
docker rm turn-srv

# 2. Xóa Image cũ trên Cloud để ép nó tải bản mới nhất về
docker rmi nguyen13104/my-turn-server:latest

# 3. Chạy lại container với Image mới vừa push
docker run -d \
  --name turn-srv \
  --net=host \
  --restart always \
  nguyen13104/my-turn-server:latest

# 4. Kiểm tra xem Dashboard đã hiện ra chưa
docker logs -f turn-srv
```

---

## LƯU Ý QUAN TRỌNG:
Sau khi bạn đã thực hiện xong Bước 2 (Cloud đã chạy code mới), hãy nhắn tôi để tôi **đổi lại file stunServers.js trong Frontend** sang chế độ dùng tên phòng.
